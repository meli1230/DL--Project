# Deep Learning Project

## Work environment
- `Google Colab` with `Python`
- scientific paper written in `Latex (Overleaf)` in `Springer LNCS` format
- From `SemEval 2026`, Task 13: Detecting Machine-Generated Code with Multiple Programming Languages, Generators, and Application Scenarios


## Challenge structure
- **main task:** detecting machine-generated code from human-written code across difference programming languages, domains and generation techniques
- **task A:** 
	- `fully human-written` / `fully machine-generated`
	- training languages: `C++`, `Python`, `Java`
- **task B:** 
	- `multi-class authorship detection`
	- human / one of 10 LLM families (`DeepSeek`, `Qwen`, `01-ai`, `BigCode`, `Gemma`, `Phi`, `Meta-LLaMA`, `IBM-Granite`, `Mistral`, `OpenAI`)
      - target metric: `macro-f1-score`
- **task C:** 
	- `human-written` / `machine-generated` / `hybrid` (partially written by LLM) / `adversarial` (generated via adversarial prompts or RLHF to mimic humans)
	- target metric: `macro-f1-score`


## Research questions
- **Main Q:** How effectively can transformed based models distinguish machine-generated code from human-written code across multiple programming languages?
- **q1:** How well do deep learning models generalize to unseen programming languages and unseen domains?
- **q2:** Can deep learning models distinguish between code written by humans and code generated by different LLM families?
- **q3:** Can deep learning models effectively detect hybrid code where human and LLM contributions are interleaved?


## Dataset
- large-scale dataset 	
- **task A:**
  - training: `500k samples` (`238K Human-Written` | `262K Machine-Generated`)
  - validation: `100k samples`
- **task B:**
  - training: `500k samples` (`442K Human |4K DeepSeek-AI` | `8K Qwen` | `3K 01-ai` |`2 K BigCode` | `2K Gemma` | `5K Phi` | `8K Meta-LLaMA` | `8K IBM-Granite` | `4K Mistral` | 
    `10K OpenAI`)
  - validation: `100k samples`
- **task C:**
  - training: `900k samples` (`485K Human-written` | `210K Machine-generated` | `85K Hybrid` | `118K Adversarial`)
  - validation: `200k samples`
- it is structured and pre-processed (code is `tokenized`, `labeled` and `consistent`)
- it is multi-lingual and multi-domain (as presented in the data split above)
- it's public and reproducible (available via Kaggle / Hugging Face)
- each sample includes
	- the `code` itself
	- a `label` (such as human, hybrid, machine or a specific LLM family)
	- the `language`
	- the `generator source` (which model produced it)


## Methods used

### Overall
| Subtask                       | Machine Learning                        | Deep Learning                                   | Advanced DL / Transfer Learning |
|-------------------------------|-----------------------------------------| ----------------------------------------------- | ------------------------------- |
| **A** (Binary)                | Logistic Regression, SVM, Decision Tree | Dense NN, CNN, BiLSTM                           | CodeBERT / CodeT5               | 
| **B** (Multiclass, 11 labels) | Multiclass LR, SVM, Random Forest       | Dense NN (Softmax), CNN, BiLSTM                 | CodeBERT / StarCoder            |
| **C** (4-class )              | Multinomial LR, SVM, XGBoost            | Dense NN (Softmax), CNN+Dense, BiLSTM+Attention | CodeT5 / CodeBERT               |


### Task A
| Method                                       | Type | Motivation                                                             |
|----------------------------------------------|------|------------------------------------------------------------------------|
| **Logistic Regression**                      | ML   | Fast, interpretable baseline for binary text/code classification       |
| **Linear SVM**                               | ML   | Strong baseline for high-dimensional sparse data                       |
| **Decision Tree / Random Forest**            | ML   | Tests whether simple structural features suffice                       |
| **Dense Neural Network (2 hidden layers)**   | DL   | Classic binary classifier                                              |
| **1D Convolutional Neural Network (1D-CNN)** | DL   | Captures local syntax patterns; analogous to n-gram filters            |
| **Bidirectional LSTM (RNN)**                 | DL   | Models long-range dependencies in code sequences                       |
| **Transfer Learning (CodeBERT / CodeT5)**    | DL   | Implements transfer learning; tests generalization to unseen languages |

### Task B
| Method                               | Type | Motivation                                                               |
|--------------------------------------|------|--------------------------------------------------------------------------|
| **Multiclass Logistic Regression**   | ML   | Multiclass baseline; interpretable coefficients per author               |
| **Linear SVM (One-vs-Rest)**         | ML   | Tests scalability of shallow models to multiclass problems               |
| **Random Forest**                    | ML   | Captures non-linear author patterns; used for comparison                 |
| **Dense NN**                         | DL   | Seen in DL Classification - Reuters newswire                             |
| **1D-CNN over Token Embeddings**     | DL   | Detects stylistic cues (naming, indentation, structure)                  |
| **Bidirectional LSTM**               | DL   | Learns author-specific sequential structure (such as comment placement)  |
| **CodeBERT / StarCoder Fine-Tuning** | DL   | Tests if embeddings capture stylistic  fingerprints                      |


### Task C

| Method                                | Type | Motivation                                                      |
|---------------------------------------|------|-----------------------------------------------------------------|
| **Logistic Regression (Multinomial)** | ML   | Establishes baseline for 4-class classification                 |
| **SVM (One-vs-Rest)**                 | ML   | Provides robustness benchmark against deep models               |
| **Gradient Boosted Trees (XGBoost)**  | ML   | Tests non-linear interactions                                   |
| **Dense NN (Softmax, 4 outputs)**     | DL   | Classic neural baseline for multi-class code classification     |
| **CNN + Dense Hybrid**                | DL   | Learns local + global representations                           |
| **BiLSTM + Attention**                | DL   | Captures long dependencies and focuses on discriminative tokens |
| **CodeT5 or CodeBERT Fine-Tuned**     | DL   | Tests robustness to hybrid/adversarial code                     |
